# Implementation Plan: Real API Integration

**Branch**: `007-api-integration` | **Date**: 2026-03-25 | **Spec**: [spec.md](spec.md)

## Summary

Replace all placeholder/hardcoded data sources in the Glowy Wallpapers Flutter app with real backend API calls across three endpoints: Bootstrap (API-1, `GET /api/v1/mobile/apps/{appId}`), Category Content (API-2, paginated wallpapers/videos), and Classifications (API-3, bento grid data). Introduces a new `app` feature for bootstrap domain/data, updates `WallpaperModel` and `CategoryModel` to match actual API field shapes, registers a public Dio instance (no auth interceptor), migrates `HomeDrawer` and `ContentPage` to use API-sourced metadata, and delivers a full unit-test suite covering all three API flows.

## Tech Stack Decisions

### Public Dio Instance

Register a second Dio instance in `InjectionContainer` without `AuthInterceptor`:

```dart
// Named registration
getIt.registerLazySingleton<Dio>(
  () => Dio(BaseOptions(
    baseUrl: AppConfig.baseUrl,
    connectTimeout: const Duration(seconds: 30),
    receiveTimeout: const Duration(seconds: 30),
  ))
  ..interceptors.add(kDebugMode ? LoggingInterceptor() : ...) ,
  instanceName: 'publicDio',
);
```

All three public API data sources inject `@Named('publicDio') Dio dio`.

### Stale-While-Revalidate Pattern (Bootstrap)

```
AppRepositoryImpl.getAppData():
  cached = localDataSource.getAppMetadata()
  if (cached != null):
    emit Left(cached) immediately
    fire background fetch в†’ on success: save to local, call onMetadataRefreshed()
    return Right(cached)
  else:
    result = await remoteDataSource.getAppData()
    if success: save to local, return Right(entity)
    if failure: return Left(failure)
```

### HomeCubit Initialization Change

```
Before: loadCategories() в†’ GetCategories use case в†’ categories only
After:  loadAppData()    в†’ GetAppData use case    в†’ AppMetadataEntity (contains categories + metadata)
```

Auto-select first category immediately after `appMetadata` is set in state.

### Model Field Replacement Strategy

`WallpaperModel` requires a **full field replacement** (not additive) because old fields (`title`, `imageUrl`, `videoUrl`, `isPremium`, `classificationIds`) do not exist in the real API. Run `dart run build_runner build --delete-conflicting-outputs` after changes.

### Pagination `hasReachedEnd` Change

```dart
// Old: PaginatedResponse.hasMore (server-computed)
// New: computed in HomeCubit
bool get hasReachedEnd => state.currentPage >= totalPages
```

`PaginatedResponse` stores `totalPages`; `HomeCubit` computes the gate condition.

## Implementation Phases

### Phase 1: Setup & Infrastructure (no feature code)
- Add `appId` to `AppConfig`
- Register `publicDio` in `InjectionContainer`
- Update `PaginatedResponse` model (`totalPages`, remove `hasMore`)

### Phase 2: New `app` Feature (bootstrap)
- `AppMetadataEntity`, `AppRepository` contract, `GetAppData` use case (domain)
- `AppMetadataModel`, `BootstrapRemoteDataSource`, `BootstrapLocalDataSource`, `AppRepositoryImpl` (data)
- Register in DI container

### Phase 3: Updated Models (categories + wallpapers)
- `CategoryModel`: add `imageCount`, remove `thumbnailUrl`
- `ClassificationModel`: `wallpaperCount` в†’ `itemCount`
- `WallpaperModel`: full field replacement
- Run `build_runner` after each model change

### Phase 4: Updated Data Sources & Repositories
- `CategoryRemoteDataSource`: new URL, remove `getCategories()`, add `appId` param
- `WallpaperRemoteDataSource`: new URL, merged `classificationId?` param, rename `perPage`в†’`limit`
- `WallpaperRepository` contract: update signature
- `WallpaperRepositoryImpl`: align with new contract

### Phase 5: Home Presentation Layer
- `HomeState`: add `appMetadata: AppMetadataEntity?` field, regenerate Freezed
- `HomeCubit`: add `GetAppData` dependency, `loadAppData()` replaces `loadCategories()`, auto-select first category
- `HomeDrawer`: read dynamic data from `HomeCubit` state
- `ContentPage`: read `about`/`privacyPolicy`/`termsOfUse` from `HomeCubit` state

### Phase 6: Test Suite
- JSON fixture files
- Data source tests (bootstrap, wallpapers, classifications)
- Repository tests (bootstrap stale-while-revalidate, wallpaper error paths)
- Use case tests (`GetAppData`)
- `HomeCubit` tests (auto-select, pagination, loadMore stop)

### Phase 7: Polish & Verification
- `dart run build_runner build --delete-conflicting-outputs`
- `flutter analyze` в†’ zero warnings
- `dart format .`
- Manual smoke test: cold start, category browse, scroll pagination
